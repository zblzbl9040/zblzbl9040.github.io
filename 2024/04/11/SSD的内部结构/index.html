SSD的内部结构

SSD的内部结构

SSD主要由控制单元(SSD控制器、主机接口、DRAM等)和存储单元(主要是NAND FLASH闪存颗粒)构成。数据通过接口进入主控，经主控中转调配后存储到各个闪存颗粒中。

三大组成部分

主要有三大部分组成：与Host对接的界面(Host interface), 闪存转换层FTL以及闪存对接界面(Flash interface)。

Host interface采用的协议包括：

  • SATA
  • PCIe/NVMe SSD协议

闪存转换层FTL是主控的核心部分，主要实现：

  • 磨损平衡（Wear-leveling）
  • 垃圾数据回收（Garbage Collection）
  • 坏块管理（Bad Block Management）
  • 数据纠错处理（ECC, Error Correction Code）

Flash interface包括了SSD主控的flash controller，用来控制主控与NAND闪存之间的数据传输。

NAND闪存是数据归属地，NAND颗粒的存在，赋予了SSD储存数据的使命。为了提高存储的性能，主控会通过不同的通道（Channel）操作NAND闪存。

NAND flash在分布式存储中的应用

  • 高性能缓存：用作读写缓存
  • 元数据存储：用作存储系统元数据
  • 日志存储：存储事务日志、大量的日志信息
  • 全闪存阵列：使用全NAND闪存的存储阵列
  • 混合存储系统：使用NAND存储热数据，HDD存储冷数据
  • 大数据处理：NAND闪存加速数据读写，提升数据处理效率
  • 分布式文件系统：使用NAND闪存提升MDS的性能

主流研究方向

(1) 性能优化

全SSD用做分布式存储系统中的存储介质，应该在系统层面怎么优化能够使SSD的性能发挥到最大？

系统(软件)层面的开销会影响硬件层面的开销，因此需要在系统层面针对SSD的特性进行优化，使得SSD的性能尽可能的发挥最大。

主要优化方式就4种：

  1. 将随机写改成顺序写 √
  2. 增大并行任务量
  3. 多个事务合并成一个事务进行处理 √
  4. 冷热数据分离 √

(2) 故障处理

使用SSD用作分布式存储的存储介质时，固态硬盘出现故障应该怎么解决？(预测or优化使故障率降低)

优化设备层，降低硬盘故障率。

NAND颗粒的内部结构

Page是闪存存储的最小读写单元、Block是闪存存储的最小擦除单元。

Cell能存储数据的原理

在存储数据时，通过施加电压，让电子穿过隧穿层达到浮栅层，根据浮栅层的电子数量来判断是’0’还是’1’。

浮栅晶体管能存放多大的数据取决于对浮栅层中电子数量的粒度划分。

寿命有限的原因是通过施加电压让电子反复穿过隧穿层，会破环隧穿层的结构，导致不能再有效的保持浮栅层中的电荷。

存储类型

  • SLC：单层单元，每个单元存储位数(1bit)，电压只有两种状态(0和1)
  • MLC：多层单元，每个单元存储位数(2bit)，电压有四种状态(00,01,10,11)
  • TLC：三层单元，每个单元存储位数(3bit)，电压有八种状态(000,001,010,011,100,101,110,111)
  • QLC：四层单元，每个单元存储位数(4bit)，电压有十六种状态(0000,0001,……,1111)

写放大

写放大：指的是在执行写入操作时，SSD硬盘发生的写入量，实际要远大于实际写入的数据量。

原因：NAND数据写入的方式是以页面为单位写入，但是要想删除数据却要以块为单位。

降低写放大的方法

  • 磨损均衡：将写操作均匀分布到各个块上，避免某些块过度磨损
  • 垃圾回收：及时回收无效数据，腾出空间
  • 预留空间：保留部分空间作为缓冲区

磨损均衡原理

  • 动态磨损均衡：更多的使用擦除次数较少的块
  • 静态磨损平衡：将冷数据迁移到磨损程度较高的块上，使磨损均匀分布。为了尽量减少块之间的磨损差异，额外的开销以迁移的形式增加。

SSD和HDD的关键区别

除了吞吐量和延迟外，HDD会将来自主机的多个请求序列化到单个队列中，每次处理一次，而SSD可以同时处理多个请求，因为SSD将多个芯片连接到多个管道，虽然每个管道一次只能处理一个请求，但多个通道的存在允许同时访问不同的芯片。因此SSD的并发性能优势不可忽略。
